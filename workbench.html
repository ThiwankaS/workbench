Install & Setup

One-command bootstrap from the repo root:

bash ./scripts/bootstrap.sh

Installs Neovim deps, copies config, runs :Lazy sync, Treesitter c/cpp, font, and optionally the snap backend.

Two installers — do not confuse them

WhatCommandInstalls
Neovim plugins:Lazy / :Lazy syncLua plugins under stdpath('data')/lazy/
CLI tools:Mason → i on a packageclangd, stylua, pyright, … on your system

Mason “Installed (0)” means no Mason packages yet — not missing Lazy plugins.

After first launch

:Mason
  :checkhealth

Install or verify Mason tools:

clangdclang-formatlua-language-server typescript-language-serverpyrightbash-language-server marksmanstyluacodelldb

C / C++ & Embedded

:Godbolt
Compiler Explorer
:GodboltRun
Run on selection

CMake compile database

cmake -S . -B build -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
  ln -sf build/compile_commands.json ./compile_commands.json

ESP-IDF / FreeRTOS

  1. . $IDF_PATH/export.sh
  2. idf.py set-target esp32s3 && idf.py build
  3. Symlink compile_commands.json to project root
  4. Add project .clangd with flag Remove: list
  5. In ~/.bashrc:
    export CLANGD_QUERY_DRIVER="$HOME/.espressif/tools/**/bin/*-gcc,$HOME/.espressif/tools/**/bin/*-g++"
  6. :lsp restart clangd

Troubleshooting

Treesitter parser missing

:TSInstall c cpp
  :TSUpdate

Needs on PATH: tree-sitter, curl, tar, C compiler.

C++ completion weak / no class members

Generate and symlink compile_commands.json (see C/C++ section).

snap.nvim playwright-core error

Re-run scripts/install_snap_backend.sh — install must include bin/node_modules/.

Theme / syntax colors

Edit roles in lua/config/theme.lua (vim.theme.syntax), restart Neovim.

Discover keys anytime

Press Space and pause — Which-key shows available bindings.
